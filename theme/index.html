A few notes about writing and running software.

Laptop

I develop on an laptop with an Apple M1 chip.

laptop.sh symlinks dotfiles to $HOME, sets up Z shell, programming language runtimes (Go, Ruby), text editor (Vim), and language servers (HTML, SQL).

Homebrew manages most packages but I use Postgres.app for Postgres and ASDF for Ruby versions.

Git

I love a monorepo.

Configure gitignore not only for Git but also ag and fzf searches.

Vim

My text editor is Neovim in a Kitty terminal. Read how to wrap text, sort lines, search a project, or spell check in Vim. I run tests and single file programs via the Leader key mapped to Space.

In Vim, I format SQL, run SQL and debug slow Postgres queries.

Postgres

I often restore my development Postgres from a production copy.

I've had good luck the last two years operating a Postgres-backed queue worked by the Ruby Postgres driver.

Speed up Postgres full-text searches with a tsvector column as cache and trigger to keep the cache updated.

Create indexes concurrently to avoid locking writes to a table.

Use concat_ws instead of concat() to avoid ugly output.

Testing

To help see unit test cause and effect "at a glance", I structure them as four-phase tests and avoid mystery guests.

I use stubs and spies and avoid mocks.

CI/CD

Daily deploys with Deno and GitHub Actions scheduled workflows

Heroku

When my Heroku production app processes change state, I send Slack notifications with AWS Lambda.

Heroku Postgres has a great feature they call "dataclips" which let users enter SQL queries into a textarea, run them against a read replica, and export to CSV. I wrote a Heroku dataclips clone for fun.

Configure Heroku Postgres read replicas with Rails to scale read traffic and free primary database connections for writes.

Fly.io

Even better is to serve a web app from multiple Fly.io regions and route read-only queries to same-region Postgres replicas.

When to access an API with a specialized library and when to write a custom thin client

End-to-end with a kanban board, GitHub code review, continuous integration, and continuous delivery

Define Unix processes in a manifest named Procfile and use tools to manage those processes
